🚀 Stall Reservation Service

A microservice responsible for managing stall reservations, availability, and booking operations within the Stall Reservation System.

📌 Overview
This project is built using Microservice Architecture with:

🔐 Auth Service (User management & JWT generation)
🌐 API Gateway (JWT validation & request routing)
🛡️ Secure communication between services
📦 Standardized API responses

The system provides secure authentication and role-based authorization using JWT (JSON Web Token).

🏗️ Architecture

Client (Frontend)
        ↓
API Gateway (JWT Validation)
        ↓
Auth Service (User Management & Token Generation)

🔐 Security Design

✅ JWT is generated by Auth Service during login
✅ API Gateway validates JWT for all protected endpoints
✅ Gateway forwards identity headers:
  X-User-Email
  X-User-Role
✅ Downstream services use headers for authorization rules

🔑 Authentication Flow

1️⃣ Register (Public Endpoint)
POST /auth/register
User creates an account.

2️⃣ Login
POST /auth/login
Returns:

{
  "token": "jwt_token_here"
}

3️⃣ Store Token (Frontend)

Store JWT in:

LocalStorage (for development)
HttpOnly Cookie (recommended for production)

4️⃣ Access Protected Endpoint

Frontend sends:

Authorization: Bearer <JWT_TOKEN>
API Gateway validates token before forwarding request.

👤 Roles & Authorization

👨‍💼 ADMIN → Can manage users
👤 USER → Limited access

Authorization is enforced using:

Spring Security

Role-based access control
