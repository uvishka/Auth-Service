Auth Service + API Gateway (JWT)
Overview

This project uses Microservice Architecture:

API Gateway validates JWT on every protected request.

Auth Service manages:

User registration

Login (JWT generation)

User management (Admin/User authorization rules)

Security Design

✅ JWT is generated by Auth Service during login
✅ JWT is validated by API Gateway for every protected endpoint
✅ Gateway forwards user identity headers to downstream services:

X-User-Email

X-User-Role

Auth Service uses these headers for authorization decisions.

Base URLs
API Gateway (Recommended for frontend calls)
http://localhost:8086
Auth Service (Internal Service)
http://localhost:8080
Common Response Format (ApiResponse)

All endpoints return the following structure:

{
"success": true,
"message": "Some message",
"data": {}
}
Error Example
{
"success": false,
"message": "Error message",
"data": null
}
Authentication Flow (Frontend)
1) Register User (Public Endpoint)

Frontend calls the registration API.

2) Redirect to Login Page

User logs in.

3) Login Returns JWT

Frontend stores the token securely (recommended: in memory or secure storage).

4) Send JWT on Every Request

Frontend must include this header:

Authorization: Bearer <JWT_TOKEN>
API Gateway Rules
Public Endpoints (No Token Required)

POST /api/auth/registeruser

POST /api/auth/login

Protected Endpoints (Token Required)

All other endpoints require JWT.

If token is missing or invalid:

401 Unauthorized → Missing or invalid format

403 Forbidden → Invalid signature / expired token / verification failed

API Endpoints
1) Register User (PUBLIC)
   Request
   POST /api/auth/registeruser

Gateway URL:

POST http://localhost:8086/api/auth/registeruser
Headers
Content-Type: application/json
Body
{
"email": "user@example.com",
"password": "123456",
"companyName": "Example Company",
"contactNumber": "0771234567",
"owner": "Company Owner"
}
Success Response (200)
{
"success": true,
"message": "User registered successfully. Please login.",
"data": {
"message": "User registered successfully. Please login.",
"user": {
"id": 15,
"email": "user@example.com",
"role": "USER",
"companyName": "Example Company",
"contactNumber": "0771234567",
"owner": "Company Owner"
}
}
}
Error Cases
Email Already Exists
{
"success": false,
"message": "Email already registered: user@example.com",
"data": null
}
Validation Error (400)

Example (short password):

{
"password": "Password must be at least 6 characters"
}

4) Get User By ID (ADMIN or Self)
   Request
   GET /api/auth/users/{id}

Example:

GET http://localhost:8086/api/auth/users/10
Headers
Authorization: Bearer <TOKEN>
Authorization Rules

ADMIN → Can access any user

USER → Can access only their own profile

Success Response (200)
{
"success": true,
"message": "User fetched successfully",
"data": {
"id": 10,
"email": "user@example.com",
"role": "USER",
"companyName": "Example Company",
"contactNumber": "0771234567",
"owner": "Company Owner"
}
}
Error Cases
USER Accessing Another User

Status: 403

{
"success": false,
"message": "You can only access your own profile",
"data": null
}
Missing Forwarded Headers

Status: 401

{
"success": false,
"message": "Unauthorized (missing X-User-Role)",
"data": null
}
